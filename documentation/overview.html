MisTiGriD documentation overview

Prerequisites
The reader is supposed to be familiar with the base concepts of autonomic computing, and Java technologies such as the Felix and iPOJO frameworks. MisTiGriD is composed of several OSGi modules compiled with Maven and running on a Felix platform.

Parts of MisTiGriD are written in Scala, basic knowledge of which could be necessary in case you end up down there (which is a nice occasion to discover this cool language).

Overview
MisTiGriD is a smart micro-grid simulation platform: an instance of MisTiGriD simulates a smart house with several appliances and their autonomic managers. MisTiGriD provides an interactive and extensible demonstration playground for developpers of autonomic systems.

MisTiGriD modules are organised as follows:
  • the model is the core library of the simulation: this is where simulated systems (electrical grid, appliances, rooms...) and their autonomic managers are implemented.
  • the view allows the user to monitor and interact with model objects in a running simulation. It consists in disk traces for post-simulation analysis and a web interface for live interaction with the simulated house.
  • the deploy library defines a simple Scala DSL for the user to specify the model objects to be run in the simulation: shape and size of rooms, lamp and heater characteristics, manager policies, etc. Some of these configurations can be later changed while the simulation is running, via the web interface.
  • a set of utility modules, including a series of third-party libraries (cf. bundles folder in Felix) and a home-made wrapper for the Akka actor library.
A typical use of MisTiGriD consists in writing an initial house configuration, running the simulation and playing with it through the web interface, and then analysing the replay through persisted traces on the disk.

Setup
Running MisTiGriD only requires a Java JRE to be installed (tested with Java 6 and 7).

Compiling MisTiGriD requires Maven 2. Run "mvn clean install" from the root folder to build MisTiGriD. Run "mvn clean javadoc:javadoc" before committing.

Eclipse users will benefit from installing the following plugins:
  • Eclipse EGit
  • Maven Eclipse plugin
  • Scala Eclipse plugin
  • Maven integration for Scala
  • Stan4j (not compulsory, but this is a neat analysis tool when diving into someone else's code)
